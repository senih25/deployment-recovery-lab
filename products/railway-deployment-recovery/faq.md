# FAQ

## Is this an official Railway product?

No. This is an independent recovery resource built from real deployment
incidents and public Railway documentation.

## Who is this for?

Founders and developers who use Railway and need a faster path from deploy
failure to root cause.

## Is this a full DevOps service?

No. The audit is intentionally narrow and focused on Railway deployment
recovery.

## What kinds of incidents are covered?

- install and lockfile failures
- wrong root or missing `package.json`
- invalid custom Railpack overrides
- `502` / app-not-responding issues
- missing start command problems

## Why not just build a tool?

Because the first goal is to validate repeated demand with real users and real
incidents before investing in automation.

## What should I prepare before asking for help?

- deployment link
- build logs
- current config files
- note on whether Railway URL or custom domain is working
