# Railway Deployment Recovery Publish Checklist

Bu checklist, public repo veya landing yayına alınmadan once son kontrol icin kullanilir.

## 1. Scope freeze

- Urun adi sabit: `Railway Deployment Recovery`
- Ilk format sabit: `free checklist + paid audit`
- Tool, CLI, GitHub Action veya SaaS vaadi yok
- Kapsam deploy/config/runtime baslangic sorunlariyla sinirli

## 2. Public content review

- `README.md` urunu 30 saniyede anlatabiliyor
- `checklist.md` jargon kullanmadan uygulanabilir
- `case-study.md` kanitli ve abartisiz
- `audit-offer.md` kim icin / ne dahil / ne haric net
- `landing-copy.md` hard-sell degil, trust-first
- `faq.md` policy, risk ve teslimat sorularini cevapliyor

## 3. Claim control

- "Railway partner" veya resmi ortaklik imasi yok
- "Guaranteed fix" gibi garanti dili yok
- "Every Railway issue" gibi fazla genis claim yok
- Kanitlanmayan teknik cikarimlar kesin ifade edilmiyor
- `partial runtime` varsa "apparent" veya "observed confusion" dili kullaniliyor

## 4. Policy safety

- Railway Station / Bounties icin promo metni hazirlanmiyor
- Public repo icerigi spam veya topluluk ici reklam mantigina dayanmiyor
- GitHub README kendi repo/yuzeyi icin ilgili ve olculu kaliyor
- Upwork/Fiverr/Contra copy'leri platform-disi yonlendirme zorlamiyor

## 5. Offer readiness

- Audit giris materiali net:
  - repo linki
  - deploy loglari
  - hata ekrani
  - env/build bilgisi
- Teslimat net:
  - root-cause summary
  - minimum safe fix plan
  - verification checklist
- Out of scope net:
  - tam refactor yok
  - full DevOps rebuild yok
  - 24/7 support yok

## 6. Lead path

- Tek ana CTA var
- CTA kullaniciyi kafa karistiran 3-4 secenege bolmuyor
- Contact path hazir:
  - email
  - form
  - Upwork profile
- Ilk temas icin istenecek minimum bilgi listesi hazir

## 7. Proof path

- Case study checklist ve audit offer'a baglaniyor
- Checklist audit offer'a baglaniyor
- README ilgili public dokumanlara link veriyor
- Kullanici urunu anlamak icin baska repo gezmek zorunda kalmiyor

## 8. Release decision

Yayin ancak su durumda yapilir:

- Public metinler tutarli
- Teknik claimler kanitli
- CTA ve iletisim yolu hazir
- Kanal sirasi net
- "Ne satiliyor?" sorusu 1 cumlede cevaplanabiliyor
